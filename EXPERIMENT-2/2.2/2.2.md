# Project - UNION ALL

## Problem Summary

XYZ Pvt Ltd maintains employee records in two tables:

- **employee** – Contains all full-time employees and some active part-time employees.
- **pt_employee** – Contains all part-time employees, including both active and inactive workers.

Write a query to combine the employee names from both tables **without removing duplicate names**.

## Solution

```sql
SELECT emp_name
FROM employee

UNION ALL

SELECT emp_name
FROM pt_employee;
```

## Explanation

- The first query retrieves all employee names from the `employee` table.
- The second query retrieves all employee names from the `pt_employee` table.
- `UNION ALL` combines the results while **preserving duplicate rows**.

## SQL Concepts Used

- SELECT
- UNION ALL

## Difference Between UNION and UNION ALL

| UNION | UNION ALL |
|--------|-----------|
| Removes duplicate rows | Keeps duplicate rows |
| Slower due to duplicate checking | Faster because no duplicate checking is performed |
| Used when unique results are required | Used when all records, including duplicates, are required |

## Time Complexity

- **UNION ALL:** Approximately **O(n + m)**, where `n` and `m` are the number of rows in the two tables.
- It is generally faster than `UNION` since it does not perform duplicate elimination.
