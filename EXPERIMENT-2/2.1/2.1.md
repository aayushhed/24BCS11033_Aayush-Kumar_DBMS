# SQL UNION

## Problem Summary

Combine the data from the **Arts** and **Science** tables using the `UNION` operator.

The `UNION` operator stacks the rows from one table on top of another while automatically removing duplicate rows.

### Rules for UNION

- Both queries must return the same number of columns.
- Corresponding columns must have compatible data types.
- Duplicate rows are removed by default.

## Solution

```sql
SELECT *
FROM Arts

UNION

SELECT *
FROM Science;
```

## Explanation

- The first `SELECT` retrieves all rows from the **Arts** table.
- The second `SELECT` retrieves all rows from the **Science** table.
- `UNION` combines the results into a single output and removes any duplicate rows.

## SQL Concepts Used

- UNION
- SELECT

## Time Complexity

- Depends on the database engine.
- Additional processing is required to remove duplicate rows, making `UNION` generally slower than `UNION ALL`.

## Note

If you want to keep duplicate rows, use `UNION ALL` instead of `UNION`.
