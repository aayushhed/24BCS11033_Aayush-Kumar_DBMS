# EXCEPT

## Problem Summary

A supermarket database contains two tables:

- **fruit** – Stores the names of all fruits available in the market.
- **inventory** – Stores the names of all items currently available in the supermarket.

Write a query to display the names of fruits that are present in the **fruit** table but **not** in the **inventory** table.

## Solution

```sql
SELECT f_name
FROM fruit

EXCEPT

SELECT inv_name
FROM inventory;
```

## Explanation

- The first query retrieves all fruit names from the `fruit` table.
- The second query retrieves all item names from the `inventory` table.
- `EXCEPT` returns only those fruit names that exist in the first result set but not in the second.
- Duplicate rows are removed automatically.

## SQL Concepts Used

- SELECT
- EXCEPT

## Difference Between UNION, UNION ALL, INTERSECT, and EXCEPT

| Operator | Description |
|----------|-------------|
| **UNION** | Combines two result sets and removes duplicate rows. |
| **UNION ALL** | Combines two result sets and keeps duplicate rows. |
| **INTERSECT** | Returns only the rows common to both result sets. |
| **EXCEPT** | Returns rows from the first result set that are not present in the second result set. |

## Time Complexity

- Depends on the database engine and indexing.
- The database compares both result sets to identify unique rows from the first query.
